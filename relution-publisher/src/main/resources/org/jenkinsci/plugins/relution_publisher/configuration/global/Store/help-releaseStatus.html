Select the default application status that should be used when uploading an application file. This setting can be overridden on each job's configuration page.

Applications in Development are typically only available to developers. When a developer is satisfied with their work they can move the application to Review. Once an application has passed review it can then be moved to Release.

Per default, applications are uploaded to Development and must be manually moved to another status through Relution's web interface. Changing the status here allows you to skip this step, e.g. when you do not review your applications prior to release (i.e. your customers are your beta testers).

Uploading in a status other than "Development" may require additional permissions. If the configured user's role does not have permission to upload with the selected status, the upload will fail with the message "access denied (role doesn't match)".
